# 概要

SNS 機能（みんなの投稿）実装後の UI/UX 改善、デザイン刷新、およびテスト実装を行いました。
特に、ライトモードとダークモードで全く異なる世界観（Claymorphism vs Neon Noir）を表現することに注力し、ユーザー体験を向上させました。

# 変更点

## 🎨 デザイン刷新 (UI/UX)

- **デザインシステムの確立:**

  - **Light Mode:** Puni-Puni Claymorphism（柔らかい粘土のような質感、パステルカラー）
  - **Dark Mode:** Neon Noir / Synthwave（夜のネオン街、発光表現、深い紫）
  - `.cursorrules` および `docs/design_system_prompt.md` にデザインルールを策定。

- **ページごとの適用:**
  - **投稿一覧 (`/posts`):** カードデザイン、リアクションボタン、新規投稿トリガーを刷新。
  - **散歩履歴一覧 (`/walks`):** 青空をイメージしたカード（Light）と、サイバーパンクな発光カード（Dark）へ変更。
  - **全ページ共通:** 背景アニメーション（浮遊するオーブ）を `application.html.erb` に移動し、全画面で適用。背景グラデーションを `bg-fixed` にしてスクロール時の違和感を解消。

## 🐛 バグ修正・機能改善

- **投稿モーダル:**
  - ラジオボタン（天気・気分）の選択解除機能を追加（`radio_toggle_controller.js`）。
  - 文字数カウンターのダークモード対応とオーバーフロー時の警告表示を修正。
  - モーダルの開閉アニメーションと backdrop のスタイル調整。
- **リアクション機能:**
  - ポップオーバー UI の調整。

## ✅ テスト実装 (RSpec)

- **Model Spec:**
  - `Post`: バリデーション（本文・天気・気分のいずれか必須）、アソシエーション。
  - `Reaction`: バリデーション（一意性制約の厳密なテスト）、アソシエーション。
- **System Spec (`spec/system/sns_spec.rb`):**
  - 投稿一覧の表示確認。
  - 新規投稿フロー（モーダル起動〜入力〜送信〜一覧反映）。
  - バリデーションエラー時の挙動確認。
  - リアクション（いいね）の動作確認。
- **全テストパス:** 既存テストへの影響がないことも確認済み。

# 確認方法

1. `bundle install`
2. `rails db:migrate`
3. `bin/dev` でサーバー起動
4. 以下の動作を確認してください：
   - **Light/Dark モード切替:** `/posts`, `/walks` でデザインが崩れず、世界観が切り替わること。
   - **投稿:** 「今日の散歩の気分をシェアしよう...」から投稿できること。
   - **リアクション:** 「＋」ボタンからスタンプを押せること。
